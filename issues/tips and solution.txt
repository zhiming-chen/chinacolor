
关于readme和vignettes


给 README 做 “减法”：
只保留 “用户决定是否用这个包所必须的信息”，删掉所有 “需要深入学习才用得到的内容”。
把复杂内容迁移到 vignettes：
任何需要超过 3 行代码解释的功能、任何涉及参数细节的说明、任何多步骤的案例，都应该放到 vignettes 中，并在 README 末尾加一句引导：“更多详细用法见 vignettes：browseVignettes("你的包名")”。
记住一个原则：README 要 “让人 30 秒内看懂包的用途并会用最核心功能”，vignettes 要 “让人花 30 分钟能完全掌握包的所有用法”。
这样分工后，README 会变得简洁清爽，用户第一印象更好；而 vignettes 则承担起 “深度教学” 的角色，满足进阶需求，两者互补且不重复。
